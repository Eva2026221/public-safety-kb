## 正確的系統架構

```

    ↓ 匯入

```

1. 在 Ragic 建立新表格，依上方欄位設計
2. 將 Excel 直接匯入 Ragic（Ragic 支援 Excel 匯入）
3. 確認資料完整性，修正亂碼或格式錯誤

## 階段二：Ragic → 查詢系統同步（下週）

### 方案 A：手動匯出（短期可用）
1. Ragic 匯出 CSV
2. 執行轉換腳本（我來寫）：CSV → knowledgeBaseEntries.ts 格式
3. git push → Vercel 自動部署

**優點**：不需要 API 設定，立即可用  
**缺點**：每次更新需手動匯出

### 方案 B：Ragic API 自動同步（中期目標）
1. Ragic 提供 REST API
2. 建立排程：每天自動從 Ragic 拉資料 → 更新知識庫
3. 同步後自動部署

**優點**：同事在 Ragic 更新後，查詢系統自動跟進  
**缺點**：需要一些開發工作

## 階段三：知識擴充流程（上線後持續進行）

### 同事新增知識的 SOP
1. 在 Ragic 開啟知識庫表格
2. 點「新增一筆」
3. 填寫：縣市、類組、檢查項目、缺失認定、來源
4. 狀態設為「已確認」（需有確認人員 + 日期）
5. 通知負責人執行同步（方案 A）或等待自動同步（方案 B）

### 品質管制原則
- **每筆資料必須有確認人員與日期**，未確認資料不匯入查詢系統
- **有縣市差異的規定，必須分開建立**，不混用
- **來源必須填寫**：法規條文號、公文字號、或電話確認紀錄
