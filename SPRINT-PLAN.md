## 三、執行 SOP

### Step 1：從 Ragic 取出資料
1. 開啟 Ragic，進入公安申報知識庫
2. 依上方優先序篩選目標條目
3. 逐筆確認答案有效性（電話/公文/法規依據）

### Step 2：改寫為標準格式
```
**【縣市】類別　主題標題**

**問：** 使用者可能問的口語問題（1 句話）

**判斷結論：** 一句話結論

（若需清單）
1. 項目一
2. 項目二

**實務說明：** 背景說明與注意事項

**缺失認定標準：** （若適用）
**禁止事項：** （若適用）

_WKB-XXX　類組：H2　階段：書表整理　審件風險：低_
```

**編號規則：**
- WKB-XXX：全國通用知識卡（WKB-001 起）
- BS-XXXXX：書表類條目（BS + 縣市代碼，如 BS-TYNL）

### Step 3：加入知識庫
1. 開啟 `src/data/knowledgeBaseEntries.ts`
2. 在陣列最後加入新條目
3. 存檔

### Step 4：驗證
1. 用黃金測試清單中的相關查詢測試
2. 確認：能命中 ✓、結論正確 ✓、縣市不跨界 ✓
3. 若有問題，調整 keywords 或答案文字

### Step 5：部署
```bash
git add src/data/knowledgeBaseEntries.ts
git commit -m "feat: 新增 [縣市] [主題] 條目"
git push origin master
# Vercel 自動部署，1–2 分鐘後上線
```
