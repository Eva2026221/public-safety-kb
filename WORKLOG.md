### ⚠️ 尚待處理
- **測試覆蓋率為零**：目前全靠人工目視測試，改動容易造成回歸
- **知識庫擴充無 SOP**：新增條目格式不統一，難以維護
- **搜尋仍有盲區**：口語化查詢、錯字、縮寫命中率待驗證
- **使用者回饋機制缺失**：查不到資料時無法收集需求
- **知識庫覆蓋範圍不明**：哪些縣市、類組已有資料，哪些空白，未有統計

## 三、工作計畫草稿

> **核心目標：讓新進人員查詢到正確的公共安全檢查知識**

---

### 階段 A｜品質穩固（建議優先，2 週）

#### A1. 建立測試清單
- 列出 30–50 筆「黃金測試查詢」，記錄預期結果
- 每次修改後逐一驗證，確保不回歸
- 格式建議：`查詢 | 預期顯示條目 | 預期為空`

#### A2. 補強搜尋邊界情境
- 錯字容錯：「申博書」→「申報書」
- 縮寫對應：「公安申報」→「公共安全申報」
- 口語化查詢：「要附什麼」→ 附件類條目

#### A3. 使用者回饋入口
- 「找不到」畫面加入「回報缺少的問題」按鈕（送 GitHub Issue 或 Google Form）
- 讓新進人員能主動反映知識庫空白

---

### 階段 B｜知識擴充 SOP（1 個月）

#### B1. 知識條目標準格式

每筆條目應包含：

```
**【縣市】類別　主題**

**問：** 使用者可能問的問題（口語化）

**判斷結論：** 一句話結論

（若需要列清單，換行列點）

**實務說明：** 背景、例外、注意事項

**缺失認定標準：** （若適用）
**禁止事項：** （若適用）

_WKB-XXX　類組：XX　階段：XX　審件風險：高/中/低_
```

#### B2. 優先擴充範圍

依「新進人員最常遇到的問題」排序：

| 優先級 | 主題 | 估計條目數 |
|--------|------|-----------|
| 🔴 高 | 各縣市 H2 書表整理（附件清單） | 20+ |
| 🔴 高 | 常見缺失認定（安全梯、消防、電梯） | 30+ |
| 🟡 中 | 各縣市申報流程差異 | 15+ |
| 🟡 中 | 改善計畫書格式與退件原因 | 10+ |
| 🟢 低 | 法規條文查詢 | 依需求 |

#### B3. 新增條目流程

1. 內部討論確認答案正確性
2. 依格式撰寫條目
3. 加入 `knowledgeBaseEntries.ts`
4. 用測試清單驗證搜尋可命中
5. push → 自動部署

---

### 階段 C｜進階功能（視需求，1–3 個月）

- **AI 生成回答**：知識庫找不到時，呼叫 Claude API 給出推斷答案（標示「AI 推斷，請確認」）
- **知識庫後台**：提供非工程師的條目新增/編輯介面
- **使用統計**：記錄哪些查詢最常見、哪些找不到，作為知識擴充依據
- **多語支援**：繁體/簡體輸入皆可查詢

## 四、討論議題

請與你確認以下方向：

1. **測試方式**：希望建立自動測試，還是維持人工清單就好？
2. **知識擴充責任人**：由誰負責新增條目？需要非工程師也能編輯嗎？
3. **回饋機制**：「找不到」時要怎麼收集需求？GitHub Issue？Google Form？LINE？
4. **AI 回答**：是否希望在知識庫無結果時提供 AI 推斷答案？
5. **優先順序**：階段 A / B / C 的順序是否符合你的期待？
