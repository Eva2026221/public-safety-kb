## 一、目標流程

```
使用者輸入關鍵字
        ↓
   搜尋 + 分類結果
        ↓
┌───────────────────────────────────────┐
│ A. 明確  → 一句簡答 + 法規 + 展開詳細  │
│ B. 縣市不明 → 問縣市（第1問）→ A      │
│ C. 主題不明 → 問情境（第2問）→ A      │
│ D. 找不到 → 建議換字 + 聯絡主管機關   │
└───────────────────────────────────────┘
```

最多問兩個問題（縣市 → 情境），問完必給答案。

---

## 二、結果分類邏輯

搜尋後依以下規則判斷：

| 情況 | 判定 | 動作 |
|------|------|------|
| 搜尋結果 0 筆 | 找不到（D） | 建議換字 |
| 結果 1 筆 | 明確（A） | 直接給答案 |
| 結果多筆，結論完全相同 | 明確（A） | 取第 1 筆給答案 |
| 結果多筆，包含 2+ 個不同縣市 | 縣市不明（B） | 問縣市 |
| 結果多筆，縣市相同，主題不同 | 主題不明（C） | 問情境 |

---

## 三、答案卡片格式

每個答案統一格式，三層結構：

```
┌─────────────────────────────────────────┐
│  ✓ / ✗  [簡答 — 結論一句話，大字]        │
│                                         │
│  📋 依據：[法規來源，如 BS-100]           │
│                                         │
│  ▼ 展開詳細                             │
│    情境 / 實務說明 / 缺失認定 / 禁止事項  │
└─────────────────────────────────────────┘
```

---

## 四、澄清問題格式

**縣市問題（第 1 問）**

```
您的場所在哪個縣市？
[台北市] [新北市] [桃園市] [台中市] ...
（只顯示結果中實際有差異的縣市 + 其他縣市）
```

選「其他縣市」→ 顯示全國通用規定（若有），並說明各地可能不同。

**情境問題（第 2 問）**

```
您目前在哪個作業階段？
[書表整理] [現場檢查] [申報] [申報前準備]
```

只顯示結果中實際存在的選項。

## 五、找不到畫面

```
找不到相關資料

建議換個關鍵字，例如：
• 加上縣市（台北市、桃園市…）
• 加上類組（H2、U2、G類…）
• 加上情境（書表整理、現場檢查…）

或直接聯絡主管機關：
• 建管機關（各縣市建管處）
• 消防機關（各縣市消防局）
```

## 六、技術變更範圍

### 修改的檔案

**`src/components/SmartQuery.tsx`（主要改動，完整重寫）**

| 現有 | 新設計 |
|------|--------|
| `HomeView` 首頁 | 保留，微調 |
| `GuidedView` 三步驟選單 | **移除**（由動態澄清問題取代） |
| `ResultCard` 詳細大卡片 | 改為 `AnswerCard`（簡答優先，詳細可展開）|
| `ResultsView` 多筆結果列表 | 改為 `AnswerView`（單一答案）+ `ClarifyView` |
| `searchByText` 回傳多筆 + 警示 | 改為 `classifyResults` 回傳結果類型 |

新增 4 個子元件：
- `AnswerCard` — 三層格式的答案卡（簡答 / 依據 / 展開）
- `ClarifyView` — 澄清問題 + 選項按鈕
- `AnswerView` — 顯示最終答案（含返回 + 繼續追問）
- `NotFoundView` — 找不到畫面

新增狀態機：
```typescript
type Step =
  | { phase: 'home' }
  | { phase: 'answer';   entry: ParsedEntry; query: string }
  | { phase: 'clarify';  question: ClarifyQuestion; context: SearchContext }
  | { phase: 'not_found'; query: string }
```

**`src/data/knowledgeBase.ts`（小幅修改）**

新增 `searchKnowledgeFiltered(query, filters)` — 在現有搜尋後套用縣市 / 類組過濾，供澄清後重新搜尋使用。現有 `searchKnowledge` 保持不變。

### 不修改的檔案

- `src/data/knowledgeBaseEntries.ts` — 資料不變
- `api/chat.ts` — 不變
- `src/types/index.ts` — 不變
