## 問題分析

### 問題一：未指定縣市時，回傳錯誤縣市的答案（主要問題）

**重現步驟**：在首頁點擊快捷查詢「照片要合日期嗎」

**根本原因**：
- `SmartQuery.tsx` 的 `searchByText()` 呼叫 `searchKnowledge(query, 1)`，只取評分最高的 **1 筆**
- 但「照片要合日期嗎」不含縣市，所有縣市的照片規定條目（BS-100 到 BS-124）評分完全相同（均為 17 分）
- 評分相同時，以陣列順序決定勝負 → **永遠回傳台北市（BS-100：需要合日期）**

**影響**：
| 縣市 | 正確答案 |
|------|---------|
| 台北市 | 需要合日期 |
| **新北市** | **不需要合日期** |
| **桃園市** | **不需要合日期** |
| **花蓮縣** | **不需要合日期** |
| **台南市** | **不需要合日期** |
| **南投縣** | **不需要合日期** |
| **交通局** | **不需要合日期** |
| 其他縣市 | 需要合日期 |

用戶若在上述縣市拿到台北市的結論，會照錯規定操作，導致退件。

## 修改計畫

### 修改一：`searchByText` 加入縣市曖昧偵測（已執行）

**檔案**：`src/components/SmartQuery.tsx`

**邏輯**：
```
若查詢含縣市 → 取 top-1（精準答覆，行為不變）
若查詢不含縣市：
  → 取 top-5
  → 若結果中出現 2+ 不同縣市 → countyAmbiguous = true，回傳全部結果
  → 否則 → 回傳 top-1（全國通用條目，行為不變）
```

**效果**：「照片要合日期嗎」查詢後，顯示多個縣市的結果卡片（台北市、新北市、桃園市⋯），附帶警示標籤「⚠️ 各縣市規定不同，請確認所在縣市」。

**驗證**：執行 `npx tsx validate.ts` 確認準確率仍為 100%（124/124）。
